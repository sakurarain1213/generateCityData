2. 使用 is_regular_district() 判断是否为常规市辖区

### Step 3: 放大OD矩阵

**脚本文件**: `step3_estimate_actual_migration.py`

#### 输入
- Step 2输出的映射后数据:
  - `{year}_migration_symmetric_mapped.csv`
  - `{year}_migration_symmetric_od_matrix_mapped.csv`
- 常住人口真实数据: `data/resident_population_00-24.xlsx`

#### 输出
- `{year}_estimated_migration.xlsx` (放大的汇总统计)
- `{year}_estimated_od_matrix.csv` (放大的OD矩阵)

#### 核心逻辑

```
┌─────────────────────────────────────────────────────────────────┐
│ 放大公式                                                       │
├─────────────────────────────────────────────────────────────────┤
│                                                               │
│   迁移率 = 样本迁移人数 / 样本常住人口                            │
│   实际迁移人数 = 迁移率 × 实际常住人口                             │
│                                                               │
│   实际常住人口来自 Excel 文件的 "常住人口数(万人)" 列            │
│   (真实数据，不可更改)                                            │
│                                                               │
└─────────────────────────────────────────────────────────────────┘
```

#### 代码实现

```python
# 读取真实常住人口
pop_df = pd.read_excel('resident_population_00-24.xlsx')
real_pop = pop_df[pop_df['Year'] == year]['常住人口数(万人)'].values[0] * 10000

# 计算迁移率
migration_rate = sampled_migrants / sampled_total

# 放大
estimated_migrants = migration_rate * real_pop

# 对OD矩阵的每一对(O,D)也按相同比例放大
scale_factor = real_pop / sampled_total
od_matrix['count'] = od_matrix['count'] * scale_factor
```

#### 关键约束
- 放大后的总流出人数必须合理（不能超过常住人口的85%）
- OD矩阵中所有(origin, destination)对的count总和应等于origin的总流出人数

**脚本文件**: `step4_generate_top20_outflow.py`

#### 输入

#### 核心逻辑

```
┌─────────────────────────────────────────────────────────────────┐
│ Top20生成逻辑                                                 │
├─────────────────────────────────────────────────────────────────┤
│ 1. 从OD矩阵按origin_city分组                                    │
│ 2. 对每个起始城市，找出流向最多的前20个目标城市                       │

```

- To_Top1_Count, To_Top2_Count, ..., To_Top20_Count: 对应人数

## 三、关键技术问题与修复方案

### 3.1 插值算法的两层架构问题

#### 问题本质

原V16算法采用了"全国曲线 × 城市比例"的两层架构：

```
第一层（全国级）: compute_national_curve_v16()
  └─ 设定增速序列 → 生成理论全国曲线
  └─ 例如：2011增速设为3.2%

第二层（城市级）: interpolate_cities_v16()
  └─ 城市比例PCHIP插值 → 汇总得到实际全国总数
  └─ 结果：实际增速3.90%（不等于3.2%！）
```

**核心矛盾**：第一层设定的增速，在第二层被"覆盖"了。

#### PCHIP的S型曲线陷阱

当对城市占比使用PCHIP插值时：

```python
# 某城市A的占比锚点
ratios = {
    2005: 3.0%,
    2010: 5.0%,  
    2015: 4.0%
}

# PCHIP插值结果（S型曲线特性）
2010: 5.0%
2011: 5.1%  ← 反弹！超过2010
2012: 4.9%
2013: 4.6%
2014: 4.2%
2015: 4.0%
```

**问题机制**：
1. PCHIP为保证平滑性，在拐点附近产生"超调"
2. 2011年比例(5.1%) > 2010年比例(5.0%)
3. 300个城市中，假设50个出现比例反弹
4. 城市绝对值 = 比例 × 全国总数
5. 汇总后，全国总数被扭曲

**数学表达**：

```
设城市i在年份y的流出数为 C_i(y)
全国总数 N(y) = Σ C_i(y)

原方法：
ratio_i(y) = PCHIP插值
C_i(y) = ratio_i(y) × N_target(y)

实际全国总数：
N_actual(y) = Σ [ratio_i(y) × N_target(y)]
            = N_target(y) × Σ ratio_i(y)

问题：Σ ratio_i(y) ≠ 1.0 (因为PCHIP扭曲)
所以 N_actual(y) ≠ N_target(y)
```

#### 具体案例：2012-2013增速反弹

**设定的全国曲线**：
- 2012增速：2.5%
- 2013增速：1.8%（应该下降）

**实际汇总结果**：
- 2012增速：4.67%
- 2013增速：5.79%（反而上升！）

**原因分析**：
```
假设有300个城市，其中50个城市的比例PCHIP插值在2012-2013出现了反弹：
  城市B: 2012占比3.5% → 2013占比3.8% (上升)
  城市C: 2012占比2.1% → 2013占比2.4% (上升)
  ...

虽然另外250个城市比例下降，但50个城市的反弹幅度更大
汇总后：
  2013总和 = Σ[城市比例 × 全国总数]
          = (250个下降的 + 50个反弹的) × 全国总数
          = (平均比例上升) × 全国总数
结果：2013绝对值增速超过2012
```

### 3.2 修复方案：线性插值 + 递减增速序列

#### 核心思想

**关键洞察**：当使用"比例 × 全国总数"的架构时，**比例曲线的单调性直接决定了汇总后的增速特性**。

| 插值方法 | 平滑性 | 单调性 | 适用场景 |
|---------|--------|--------|---------|
| PCHIP | ✅ 一阶导数连续 | ❌ 不保证 | 绝对值插值 |
| 线性插值 | ❌ 有折点 | ✅ 严格保证 | **比例插值** |

#### 实现方案

**1. 全国级：递减增速序列**

```python
# 2005-2010: 递减增速（确保2009-2010平滑）
growth_0510 = [0.145, 0.125, 0.105, 0.085, 0.070]  # 14.5% → 7%

# 归一化到2010锚点
test_val = val_05
for r in growth_0510:
    test_val *= (1 + r)
scale = val_10 / test_val
adjusted_growth = [r * scale for r in growth_0510]

# 计算各年值
cumulative = val_05
for i, rate in enumerate(adjusted_growth):
    cumulative *= (1 + rate)
    national_totals[2006 + i] = cumulative
```

**2. 城市级：线性插值比例**

```python
from numpy import interp as np_interp

# 分段线性插值
years_0510 = [2005, 2010]
years_1015 = [2010, 2015]

# 对每个城市
for city in all_cities:
    ratios = {2005: r05, 2010: r10, 2015: r15}
    
    # 2005-2010线性插值
    for y in [2006, 2007, 2008, 2009]:
        ratio = np_interp(y, years_0510, [ratios[2005], ratios[2010]])
        city_value[y] = ratio * national_totals[y]
    
    # 2010-2015线性插值
    for y in [2011, 2012, 2013, 2014]:
        ratio = np_interp(y, years_1015, [ratios[2010], ratios[2015]])
        city_value[y] = ratio * national_totals[y]
```

**优点**：
- 城市比例严格单调（无反弹）
- 汇总后的增速趋势与全国曲线设定一致
- 计算效率高

## 五、关键公式约束汇总

### 5.1 放大比例公式
```
迁移率 = 样本迁移人数 / 样本常住人口
实际迁移人数 = 迁移率 × 实际常住人口(来自Excel)
```

### 5.2 Stay Prob公式
```
stay_prob = 1 - (outflow_count / total_count)

等价于:
outflow_count = total_count × (1 - stay_prob)
```

### 5.3 Top20约束
```
sum(To_Top{i}_Count for i in 1..20) <= Outflow_Count × 0.99

Top20覆盖率 = sum(To_Top{i}_Count) / Outflow_Count
合理范围: 80% - 95%
```

### 5.4 插值增速约束
```
2005-2010递减增速序列: [14.5%, 12.5%, 10.5%, 8.5%, 7%]
2010-2015递减增速序列: [3.2%, 2.5%, 1.8%, 1.2%]

归一化条件:
val_10 = val_05 × Π(1 + adjusted_growth_i) for i in 2006..2010
val_14 = val_10 × Π(1 + adjusted_growth_i) for i in 2011..2014
```

### 5.5 城市比例单调性
```
对于线性插值区间[year_a, year_b]:
ratio(year_i) = ratio(year_a) + (year_i - year_a) / (year_b - year_a) × [ratio(year_b) - ratio(year_a)]

保证: ratio单调 ⇔ 城市绝对值增速稳定
```
